# PeakRack Risk 升级说明

## 1.2.0

面向 WHMCS 9.x / PHP 8.3 的运维和扩展性版本。

### 新增

- 右上角后台语言切换，布局与 PeakRack Popup 模块保持一致。
- 右上角 WHMCS 活动日志镜像级别控制。
- 最近 7 天和 30 天风险指标，包含通过、审核、欺诈、白名单和平均分。
- 诊断面板：检查模块数据表、`rule_version` 迁移、WHMCS token 辅助函数、`localAPI`、`logActivity`、最近决策、最近审计和当前规则版本。
- JSON 配置导出和导入工具。
- 产品 ID 白名单和付款方式白名单。
- 信任策略：客户账号年龄、已付发票历史、已验证邮箱。
- 后台决策表和订单详情风险面板中的风险原因本地化、可读化显示。
- 针对已有模块数据表的增量结构迁移检查。

### 优化

- 底部“最近决策”和“最近审计日志”最多读取 100 行，并放入固定高度滚动区域，避免订单多时页面无限拉长。
- 自动 `AfterFraudCheck` 处理在同一规则版本已经处理过订单时跳过重复订单动作；手动工具仍可有意识地重新执行。
- 决策记录在 `rule_version` 字段可用时保存当前规则版本。
- 插件版本号升级到 `1.2.0`。

### 升级注意

- 已有设置会保留，并自动合并新默认项。
- 升级或重新启用插件后会补齐 `mod_peakrack_risk_decisions.rule_version` 字段。
- 生产环境启用新的白名单和信任策略前，建议先确认规则是否符合你的业务。
- JSON 导入会在校验和归一化后替换当前有效配置快照。

## 1.1.0

面向 WHMCS 9.x / PHP 8.3 的运营工具增强版本。

### 新增

- WHMCS 活动日志镜像级别：可选择全部审计记录、警告和错误、仅错误，或完全不镜像。
- 订单详情页风险面板，会注入到 `orders.php?action=view&id=...` 页面。
- 插件后台手动订单工具：
  - 仅重新评分
  - 按当前规则处理
  - 强制执行 `PendingOrder`
  - 强制执行 `FraudOrder`

### 优化

- 自动风控 Hook 与后台手动动作共用同一套订单处理辅助逻辑。
- 默认只把警告和错误写入 WHMCS Activity Log，普通 info 审计保留在插件自己的审计表中。
- 插件版本号升级到 `1.1.0`。

### 升级注意

- 已保存的后台设置会保留。
- 升级后建议检查新的 **WHMCS 活动日志** 设置。
- 强制动作会真实调用 WHMCS `localAPI`，生产使用前建议先用非关键订单测试。

## 1.0.1

稳定性和生产安全性优化版本。

### 新增

- 请求级配置缓存。
- 更兼容动态和自定义订单模板的结账确认逻辑。
- 服务端阈值关系归一化。
- 结构化的 `localAPI` 异常处理。
- 审计日志和规则版本保留策略。
- 每日 Cron 清理旧审计日志和规则快照。

### 优化

- 新安装默认启用“仅记录日志”模式。
- 结账确认状态绑定当前服务端 nonce。
- 重复下单和历史欺诈检查减少不必要的大表全量计数。

### 升级注意

- 已保存的后台设置会保留；更保守的默认值只影响新安装。
- 升级后建议完整测试一次结账流程，尤其是使用自定义订单模板的网站。
